===========================================
简历项目经历 — 咸虾米壁纸
===========================================

【简短版 — 适合海投，3 行讲清楚】

咸虾米壁纸（uni-app + Vue3 跨平台壁纸应用）
- 独立开发，一套代码编译到微信小程序、抖音小程序、App、H5 四端运行
- 实现壁纸浏览、分类检索、专题推荐、预览下载、评分打分、排行榜、搜索历史等 10+ 功能模块
- 封装统一请求层（错误拦截/鉴权），基于本地存储实现跨页面数据共享，条件编译适配多端差异


【标准版 — 适合面试带作品，4-5 条要点】

项目名称：咸虾米壁纸 — 跨平台壁纸应用
技术栈：uni-app（Vue 3）+ SCSS + uni-ui + RESTful API
项目描述：一款多端壁纸浏览下载小程序/App，支持推荐、分类、专题、搜索、排行榜等功能

主要工作：
- 独立负责全部前端开发，完成 12 个页面、5 个可复用组件，适配微信/抖音/App/H5 四端
- 封装两层 API 请求模块（基础请求 + 业务接口），统一处理异常、鉴权、错误提示
- 实现壁纸预览页：swiper 大图滑动、图片预加载（仅加载前后各一张）、遮罩交互、多端下载（App 调系统相册 + 权限引导，H5 降级为长按保存）
- 使用 Grid 布局实现响应式壁纸瀑布流，骨架屏优化加载体验，触底分页加载
- 封装通用工具函数（时间格式化、数字缩写、页面跳转降级策略），提高代码复用率


【精简版 — 一行流，适合简历空间很紧张时】

基于 uni-app + Vue3 开发的跨平台壁纸应用，独立完成 12 页面 5 组件，支持微信/抖音/App/H5 四端运行，包含壁纸浏览、分类、搜索、预览下载、评分、排行榜等核心功能，封装统一请求层并处理多端兼容。


【面试自我介绍时说的版本】

"我做了一个壁纸小程序，用的是 uni-app 和 Vue 3。这个项目一套代码跑在微信小程序、抖音小程序、App 和 H5 网页四个端。功能上包括了首页推荐、分类浏览、专题合集、壁纸预览下载、评分、搜索、排行榜这些。项目里我负责了全部前端开发，技术上的亮点是封装了统一的网络请求层、用条件编译处理了不同平台的差异（比如下载功能在不同端的实现完全不同）、以及预览页的图片预加载逻辑。代码量大概 12 个页面加 5 个公共组件。"
